## AutoFormProps

<AutoFormPropsTable />

UI integrations such as `@autoform/mui`, `@autoform/ant`, etc. provide default uiComponents and formComponents, while still allowing you to override them. They provide AutoForm with `ExtendableAutoFormProps` type, which is the same as `AutoFormProps`, except uiComponents and formComponents props are optional.

### `schema`

`schema` must be a schema provider instance from one of the schema packages.

```tsx
import { ZodProvider } from "@autoform/zod";
import * as z from "zod";

const schema = z.object({
  username: z.string(),
});

<AutoForm schema={new ZodProvider(schema)} withSubmit />;
```

### `defaultValues` vs `values`

Use `defaultValues` for initial form state. Use `values` only when the form should react to external state changes.

```tsx
<AutoForm schema={schemaProvider} defaultValues={{ username: "johndoe" }} />
```

```tsx
const [user, setUser] = useState({ username: "johndoe" });

<AutoForm schema={schemaProvider} values={user} />;
```

### `formControl`

Use `formControl` when a parent component needs direct access to react-hook-form methods. Create it once per form instance so the form state and subscriptions are stable across renders.

```tsx
import * as React from "react";
import { createFormControl } from "react-hook-form";

function MyForm() {
  const { formControl, reset } = React.useMemo(() => createFormControl(), []);

  return (
    <>
      <AutoForm schema={schemaProvider} formControl={formControl} />
      <button onClick={() => reset()}>Reset</button>
    </>
  );
}
```

### `formComponents`

`formComponents` maps AutoForm field types to React components. It is the right extension point for custom inputs.

```tsx
<AutoForm
  schema={schemaProvider}
  formComponents={{
    markdown: MarkdownEditor,
    colorPicker: ColorPickerInput,
  }}
/>
```

### `uiComponents`

`uiComponents` replaces structural pieces such as wrappers, error messages, and the submit button. It is the right extension point for layout and design-system integration.

```tsx
<AutoForm
  schema={schemaProvider}
  uiComponents={{
    FieldWrapper: CustomFieldWrapper,
    ErrorMessage: ({ error }) => <span className="text-red-500">{error}</span>,
  }}
/>
```

## Custom fields

Custom field components receive `AutoFormFieldProps`. In most cases, call `useField()` and spread the returned field props into your input. `useField()` accepts the same options as react-hook-form's [`useController`](https://react-hook-form.com/docs/usecontroller), except `name`, which is prefilled from the current field path.

```tsx
import { AutoFormFieldProps } from "@autoform/react";

export function StringField({ id, inputProps, useField }: AutoFormFieldProps) {
  const field = useField();

  return <input id={id} {...inputProps} {...field} value={field.value ?? ""} />;
}
```

## UI components

These components are used by the base renderer to assemble the form. Override them when you want to change layout, labels, error placement, object sections, array controls, or submit behavior.

| Component             | Purpose                                      | Example implementation                                                                                                         |
| --------------------- | -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `Form`                | Root form element.                           | [MUI Form](https://github.com/vantezzen/autoform/blob/main/packages/mui/src/components/Form.tsx)                               |
| `FieldWrapper`        | Label, error, description, and field layout. | [MUI FieldWrapper](https://github.com/vantezzen/autoform/blob/main/packages/mui/src/components/FieldWrapper.tsx)               |
| `ErrorMessage`        | Configuration error display.                 | [MUI ErrorMessage](https://github.com/vantezzen/autoform/blob/main/packages/mui/src/components/ErrorMessage.tsx)               |
| `SubmitButton`        | Default submit button for `withSubmit`.      | [MUI SubmitButton](https://github.com/vantezzen/autoform/blob/main/packages/mui/src/components/SubmitButton.tsx)               |
| `ObjectWrapper`       | Nested object field layout.                  | [MUI ObjectWrapper](https://github.com/vantezzen/autoform/blob/main/packages/mui/src/components/ObjectWrapper.tsx)             |
| `ArrayWrapper`        | Array field layout and add-item control.     | [MUI ArrayWrapper](https://github.com/vantezzen/autoform/blob/main/packages/mui/src/components/ArrayWrapper.tsx)               |
| `ArrayElementWrapper` | One array item and remove-item control.      | [MUI ArrayElementWrapper](https://github.com/vantezzen/autoform/blob/main/packages/mui/src/components/ArrayElementWrapper.tsx) |
