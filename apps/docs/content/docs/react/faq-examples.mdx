## How do I submit, reset AutoForm from buttons in a Dialog?

Use either approach:

1.  Autoform wraps the form with [`FormProvider`](https://react-hook-form.com/docs/formprovider/). Render the dialog components as `AutoForm` children and use the [`useFormContext`](https://react-hook-form.com/docs/useformcontext/) hook to access form methods and states.

2.  Access form methods and states externally with [`createFormControl`](https://react-hook-form.com/docs/createFormControl) and trigger actions like submission, reset etc.

## How do I create fields with dependencies between them?

**1. Validation** — use Zod's [refinement functions](https://zod.dev/api#refinements) (`superRefine`, `refine`, or `check`) to express cross-field rules. These run on submission and integrate with AutoForm's resolver, so error messages are placed on the correct fields automatically.

**2. UI behaviour** — Control rendering and interactivity by using React Hook Form's [`useWatch`](https://react-hook-form.com/docs/usewatch) and [`useFormContext`](https://react-hook-form.com/docs/useformcontext) inside a custom field component or a `fieldWrapper`. AutoForm wraps every form in a `FormProvider`, making these hooks work anywhere inside the tree.

| Pattern                    | Example in the demo below                                                    |
| -------------------------- | ---------------------------------------------------------------------------- |
| **Disable** a field        | Gift message — visible but disabled until _This is a gift_ is checked        |
| **Hide** a field           | Coupon code — invisible until _"Have a coupon?"_ is checked                  |
| **Conditional validation** | Card details — required by `superRefine` only when no 100% coupon is applied |
| **Dynamic options**        | State dropdown — list rebuilds every time the country changes                |

## How do I create a simple multistep form?

Use a separate schema for each step. Render the current step, call `trigger()` to validate
it, then use `getValues()` to read and save its values in state before moving forward.
Since each step is its own AutoForm instance, validation is isolated to the current step.
