This guide will walk you through the process of creating a custom UI integration for @acp-autoform/react. By following these steps, you'll be able to use AutoForm with your preferred UI library or custom components.

## Set up the project

You can create your integration as a custom npm package (e.g. "@acp-autoform/custom-ui") or as part of your existing project. For this guide, we'll create a new npm package.

```bash
mkdir @autoform-custom-ui
cd @autoform-custom-ui
npm init -y
```

Install the necessary dependencies:

```bash
npm install @acp-autoform/react react
```

## Create the main export file

Create a new file called `src/index.tsx` that will export all the necessary components and types:

```tsx
// src/index.tsx
export * from "./AutoForm";
export * from "./types";
```

## Define custom types

Create a new file called `src/types.ts`:

```tsx
// src/types.ts
import {
  AutoFormUIComponents,
  AutoFormFieldComponents,
} from "@acp-autoform/react";

export interface CustomAutoFormUIComponents extends AutoFormUIComponents {
  // Add any additional UI components specific to your integration
  // This could include theme configuration, layout components, etc.
}

export interface CustomAutoFormFieldComponents extends AutoFormFieldComponents {
  // Add any additional field components specific to your integration
}
```

## Implement UI components

Create a new folder called `src/components` and implement the required UI components. You'll need to create the following components:

1. Form
2. FieldWrapper
3. ErrorMessage
4. SubmitButton
5. ObjectWrapper
6. ArrayWrapper
7. ArrayElementWrapper

You can take a look at existing implementations like "@acp-autoform/mui" for inspiration. Here's an example of how to implement these components with pure HTML:

```tsx
// src/components/Form.tsx
import React from "react";

export const Form: React.FC<{
  onSubmit: (e: React.FormEvent) => void;
  children: React.ReactNode;
}> = ({ onSubmit, children }) => {
  return <form onSubmit={onSubmit}>{children}</form>;
};
```

```tsx
// src/components/FieldWrapper.tsx
import React from "react";
import { FieldWrapperProps } from "@acp-autoform/react";

export const FieldWrapper: React.FC<FieldWrapperProps> = ({
  label,
  error,
  children,
  id,
  field,
}) => {
  return (
    <div>
      <label htmlFor={id}>{label}</label>
      {children}
      {error && <span>{error}</span>}
    </div>
  );
};
```

```tsx
// src/components/ErrorMessage.tsx
import React from "react";

export const ErrorMessage: React.FC<{ error: string }> = ({ error }) => (
  <span style={{ color: "red" }}>{error}</span>
);
```

```tsx
// src/components/SubmitButton.tsx
import React from "react";

export const SubmitButton: React.FC<{ children: React.ReactNode }> = ({
  children,
}) => <button type="submit">{children}</button>;
```

```tsx
// src/components/ObjectWrapper.tsx
import React from "react";
import { ObjectWrapperProps } from "@acp-autoform/react";

export const ObjectWrapper: React.FC<ObjectWrapperProps> = ({
  label,
  children,
  parsedField,
}) => (
  <fieldset>
    <legend>
      {label}
      {parsedField.required && " *"}
    </legend>
    {parsedField.fieldConfig?.description && (
      <p>{parsedField.fieldConfig.description}</p>
    )}
    {children}
  </fieldset>
);
```

```tsx
// src/components/ArrayWrapper.tsx
import React from "react";
import { ArrayWrapperProps } from "@acp-autoform/react";

export const ArrayWrapper: React.FC<ArrayWrapperProps> = ({
  label,
  error,
  children,
  inputProps,
  onAddItem,
  parsedField,
}) => (
  <fieldset>
    <legend ref={inputProps.ref}>
      {label}
      {parsedField.required && " *"}
    </legend>
    {error && <span>{error}</span>}
    {children}
    <button type="button" onClick={onAddItem}>
      Add item
    </button>
  </fieldset>
);
```

```tsx
// src/components/ArrayElementWrapper.tsx
import React from "react";
import { ArrayElementWrapperProps } from "@acp-autoform/react";

export const ArrayElementWrapper: React.FC<ArrayElementWrapperProps> = ({
  children,
  onRemove,
  index,
}) => (
  <div>
    <div>Item {index + 1}</div>
    {children}
    <button type="button" onClick={onRemove}>
      Remove item
    </button>
  </div>
);
```

## Implement field components

In the same `src/components` folder, implement the field components. There is no requirement on which field components you need to implement, but you should cover the basic types like string, number, and date.

Here's an example of how to implement these components:

```tsx
// src/components/StringField.tsx
import React from "react";
import { useController } from "react-hook-form";
import { AutoFormFieldProps } from "@acp-autoform/react";

export const StringField: React.FC<AutoFormFieldProps> = ({
  inputProps,
  error,
  id,
}) => {
  const { field } = useController({ name: id });

  return (
    <input
      id={id}
      aria-invalid={!!error}
      {...inputProps}
      {...field}
      value={field.value ?? ""}
    />
  );
};
```

```tsx
// src/components/NumberField.tsx
import React from "react";
import { useController } from "react-hook-form";
import { AutoFormFieldProps } from "@acp-autoform/react";

export const NumberField: React.FC<AutoFormFieldProps> = ({
  inputProps,
  id,
}) => {
  const { field } = useController({ name: id });

  return <input id={id} type="number" {...inputProps} {...field} />;
};
```

```tsx
// src/components/DateField.tsx
import React from "react";
import { useController } from "react-hook-form";
import { AutoFormFieldProps } from "@acp-autoform/react";

export const DateField: React.FC<AutoFormFieldProps> = ({ inputProps, id }) => {
  const { field } = useController({ name: id });

  return <input id={id} type="date" {...inputProps} {...field} />;
};
```

You'll also want to implement boolean, select, and any other field types your schemas use. For example, boolean fields are commonly rendered as checkboxes, and select fields can be rendered as `<select>` elements or as components from your UI library.

Additionally, you can create custom field components for other types like checkboxes, radio buttons, etc. These can later be used by setting a custom `fieldType` in the schema.

`inputProps` contains props from `fieldConfig.inputProps`. Import `useController` from `react-hook-form` to connect the component to the form. Use `useController({ name: id })` with the `id` from `AutoFormFieldProps`.

## Create the AutoForm component

Create a new file called `src/AutoForm.tsx`. This will wrap the base `AutoForm` component from `@acp-autoform/react` and provide your custom UI components:

```tsx
// src/AutoForm.tsx
import { AutoForm as BaseAutoForm } from "@acp-autoform/react";
import {
  CustomAutoFormUIComponents,
  CustomAutoFormFieldComponents,
} from "./types";
import {
  Form,
  FieldWrapper,
  ErrorMessage,
  SubmitButton,
  ObjectWrapper,
  ArrayWrapper,
  ArrayElementWrapper,
} from "./components";
import { StringField, NumberField, DateField } from "./components";

const uiComponents: CustomAutoFormUIComponents = {
  Form,
  FieldWrapper,
  ErrorMessage,
  SubmitButton,
  ObjectWrapper,
  ArrayWrapper,
  ArrayElementWrapper,
};

const formComponents: CustomAutoFormFieldComponents = {
  // The key should match the data type (string, number, date, etc.) or a custom field type (e.g. radio)
  string: StringField,
  number: NumberField,
  date: DateField,
};

export function AutoForm<T extends Record<string, any>>(
  props: Omit<
    Parameters<typeof BaseAutoForm>[0],
    "uiComponents" | "formComponents"
  >,
) {
  return (
    <BaseAutoForm
      {...props}
      uiComponents={uiComponents}
      formComponents={formComponents}
    />
  );
}
```

## Add custom field types (optional)

If you want to add custom field types, you can create new components and add them to the `formComponents` object in `src/AutoForm.tsx`. For example:

```tsx
// src/components/CustomField.tsx
import React from "react";
import { useController } from "react-hook-form";
import { AutoFormFieldProps } from "@acp-autoform/react";

export const CustomField: React.FC<AutoFormFieldProps> = ({
  error,
  id,
  inputProps,
}) => {
  const { field } = useController({ name: id });

  return (
    <div>
      <input
        id={id}
        type="text"
        aria-invalid={!!error}
        {...inputProps}
        {...field}
        value={field.value ?? ""}
      />
      <button
        type="button"
        onClick={() => field.onChange(`${field.value ?? ""}!`)}
      >
        Add !
      </button>
    </div>
  );
};
```

Then, add it to the `formComponents` object in `src/AutoForm.tsx`:

```tsx
// src/AutoForm.tsx
// ... other imports
import { CustomField } from "./components/CustomField";

const formComponents: CustomAutoFormFieldComponents = {
  // ... other fields
  custom: CustomField,
};
```

## Package and publish (optional)

Update your `package.json` file with the appropriate information and scripts:

```json
{
  "name": "@acp-autoform/custom-ui",
  "version": "1.0.0",
  "main": "dist/index.js",
  "types": "dist/index.d.ts",
  "scripts": {
    "build": "tsc",
    "prepublishOnly": "npm run build"
  },
  "peerDependencies": {
    "react": "^17 || ^18 || ^19",
    "@acp-autoform/react": "^1.0.0"
  },
  "devDependencies": {
    "typescript": "^4.5.0"
  }
}
```

Build your package:

```bash
npm run build
```

Now you can publish your package to npm:

```bash
npm publish
```

## Using your custom UI integration

To use your custom UI integration in a project, install it alongside @acp-autoform/react or use the integration in your project directly

```bash
npm install @acp-autoform/react @acp-autoform/custom-ui
```

Then, in your React component:

```tsx
import { AutoForm } from "@acp-autoform/custom-ui";
import { ZodProvider, fieldConfig } from "@acp-autoform/zod";

const mySchema = z.object({
  // ... your schema definition
});
const schemaProvider = new ZodProvider(mySchema);

function MyForm() {
  return (
    <AutoForm
      schema={schemaProvider}
      onSubmit={(data) => {
        console.log(data);
      }}
      withSubmit
    />
  );
}
```

By following these steps, you've created a custom UI integration for @acp-autoform/react that can be used with your preferred UI components or library. You can further customize the components and add more field types as needed for your specific use case.
