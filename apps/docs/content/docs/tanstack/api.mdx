## AutoFormProps

<AutoFormPropsTable />

UI integrations such as `@acp-autoform/mui`, `@acp-autoform/ant`, etc. provide default `uiComponents` and `formComponents`, while still allowing you to override them.

### `schema`

`schema` must be a schema provider instance from one of the schema packages.

```tsx
import { ZodProvider } from "@acp-autoform/zod";
import * as z from "zod";

const schema = z.object({
  username: z.string(),
});

<AutoForm schema={new ZodProvider(schema)} withSubmit />;
```

### `defaultValues` vs `values`

Use `defaultValues` for the initial form state. Use `values` only when the form should react to external state changes.

```tsx
<AutoForm schema={schemaProvider} defaultValues={{ username: "johndoe" }} />
```

```tsx
const [user, setUser] = useState({ username: "johndoe" });

<AutoForm schema={schemaProvider} values={user} />;
```

For TanStack Form, `values` is synchronized into the current form with `setFieldValue`. This is useful for controlled external state, but `defaultValues` is simpler when you only need initial values.

### `formControl`

Use `formControl` when a parent component needs direct access to TanStack Form methods and state. Create it with `useAppForm` from `@acp-autoform/react/tanstack-form`.

```tsx
import { formOptions } from "@tanstack/react-form";
import { useAppForm } from "@acp-autoform/react/tanstack-form";

function MyForm() {
  const form = useAppForm(formOptions({ defaultValues: {} }));

  return (
    <>
      <AutoForm schema={schemaProvider} formControl={form} />
      <button onClick={() => form.reset()}>Reset</button>
    </>
  );
}
```

AutoForm owns the external form's `defaultValues` and `onSubmit` options so its props and final schema validation remain authoritative. Other options supplied to `useAppForm`, including `validationLogic`, listeners, transforms, async settings, and validators, are preserved. AutoForm adds its schema validator, default revalidation logic, and invalid-field focus behavior only where the external form did not provide an equivalent option.

### `formComponents`

`formComponents` maps AutoForm field types to React components. It is the right extension point for custom inputs.

```tsx
<AutoForm
  schema={schemaProvider}
  formComponents={{
    markdown: MarkdownEditor,
    colorPicker: ColorPickerInput,
  }}
/>
```

### `uiComponents`

`uiComponents` replaces structural pieces such as wrappers, error messages, and the submit button. It is the right extension point for layout and design-system integration.

```tsx
<AutoForm
  schema={schemaProvider}
  uiComponents={{
    FieldWrapper: CustomFieldWrapper,
    ErrorMessage: ({ error }) => <span className="text-red-500">{error}</span>,
  }}
/>
```

## Custom fields

Custom field components receive `AutoFormFieldProps`. Use `useFieldContext` from `@acp-autoform/react/tanstack-form` to connect custom inputs to the TanStack field API.

```tsx
import { AutoFormFieldProps } from "@acp-autoform/react";
import { useFieldContext } from "@acp-autoform/react/tanstack-form";

export function StringField({ id, inputProps }: AutoFormFieldProps) {
  const field = useFieldContext<string>();

  return (
    <input
      id={id}
      name={String(field.name)}
      value={field.state.value ?? ""}
      onBlur={field.handleBlur}
      onChange={(event) => field.handleChange(event.target.value)}
      {...inputProps}
    />
  );
}
```

Use the same hook when you need TanStack-specific state or methods.

```tsx
import { useFieldContext } from "@acp-autoform/react/tanstack-form";

function CustomStatus() {
  const field = useFieldContext();

  return field.state.meta.isTouched ? <span>Touched</span> : null;
}
```

## TanStack adapter exports

```tsx
import {
  AutoForm,
  useAppForm,
  useFormContext,
  useFieldContext,
} from "@acp-autoform/react/tanstack-form";
```

Use UI package subpaths for rendered forms:

```tsx
import { AutoForm } from "@acp-autoform/mui/tanstack-form";
```
